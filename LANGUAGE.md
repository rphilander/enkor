# enkor Language Design

enkor is a small Lisp-like language with Clojure-inspired immutability, implemented in plain C. It is designed to be generated by LLMs — simplicity and strict syntax minimize generation errors.

## Values

All data is represented by a single immutable type (`Val`) supporting 10 types:

- `nil` — the absence of a value
- `true`, `false` — booleans
- `42`, `-3` — 64-bit signed integers
- `3.14`, `-0.5` — 64-bit floats (must have digits on both sides of the dot, no scientific notation)
- `"hello"` — strings (opaque UTF-8 byte sequences, not indexable)
- `foo`, `my-fn?`, `+` — symbols (identifiers, following Clojure naming rules)
- `:name`, `:type` — keywords (identifiers prefixed with `:`)
- `(1 2 3)` — lists
- `{:a 1 :b 2}` — maps (any value can be a key)
- error values (not a literal form, produced by runtime errors)

## Syntax

### Atoms

```
nil                     ; nil
true false              ; booleans
42 -3 0                 ; integers
3.14 -0.5 0.0           ; floats (digits required on both sides of .)
"hello"                 ; string
"line\none"             ; string with escape sequences
foo my-fn? + *          ; symbols
:name :type             ; keywords
```

### Compound forms

```
(+ 1 2)                 ; list
()                      ; empty list
{:a 1 :b 2}             ; map (alternating key value pairs, must be even)
{}                      ; empty map
```

### Not supported

- `[brackets]` — no vectors/arrays. Use `()` for lists.
- `,` commas — not treated as whitespace, not valid syntax.
- `'quote` — no reader macros. Use `(quote x)` if needed.
- `1e10` — no scientific notation for floats.
- `.5` — no leading-dot floats. Write `0.5`.
- `42.` — no trailing-dot floats. Write `42.0`.
- Character indexing into strings.

### Whitespace and comments

Whitespace is spaces, tabs, newlines, and carriage returns. Comments start with `;` and extend to end of line.

### String escape sequences

Only four escape sequences are supported:

- `\"` — literal double quote
- `\\` — literal backslash
- `\n` — newline
- `\t` — tab

Any other escape (e.g. `\q`) is a parse error.

### Numbers

A `-` immediately followed by a digit is a negative number literal. A `-` followed by a space or a letter is a symbol. So `-42` is the integer negative forty-two, `- 42` is the symbol `-` followed by `42`, and `-foo` is the symbol `-foo`.

Integers and floats are distinct types with no implicit promotion. `42` is an int, `42.0` is a float.

### Maps

Map literals contain an even number of forms — alternating keys and values. Any value can be a key. Duplicate keys resolve as last-write-wins. Odd form count is a parse error.

## Design principles

1. **LLM-first** — the language is designed to be generated by LLMs, not written by humans. Strict syntax with clear error messages minimizes generation errors.
2. **Immutability** — all values are immutable. This makes reference counting safe (no cycles) and equality well-defined.
3. **Everything is a Val** — all APIs accept and return `Val *`. No separate type systems or out-of-band channels.
4. **Total ordering** — all values are comparable via `val_cmp`, enabling any value to be a map key.
5. **Simplicity over performance** — we use C for its small language footprint, not for speed.
